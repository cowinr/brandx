Note

All cost figures in this document are inclusive of infrastructure provisioning. Staffing is reported separately in the workforce plan.

Tip

Use the brandx init command to generate a fully-commented starter config at ~/.config/brandx/brand.yaml.

Important

The config cascade resolves lowest-to-highest: defaults → home config → frontmatter → flags. A key set in frontmatter overrides the same key in the home config.

Warning

Pinning the markdown package version is intentional. The styling layer is coupled to python-markdown's exact HTML output shape; a version bump may break rendering.

Caution

Do not commit brand configs containing personal contact information to public repositories.

Technical Reference

Config cascade (simplified)

The resolver deep-merges four layers in order. The example below shows how a single accent override in frontmatter leaves the rest of the palette intact:

# ~/.config/brandx/brand.yaml
identity:
  name: Richard Cowin
  role: Enterprise Architect

---
# document.md frontmatter
colours:
  accent: "#e63946"   # only this key overrides; palette is otherwise intact

CLI invocation

# Render a document
brandx render report.md

# Render email HTML to clipboard
brandx render --email report.md --clipboard

# Flag override (dotted key)
brandx render report.md --set colours.accent="#e63946"

The design principle is "identity is pure data, the engine is generic." The engine renders whatever resolved config it is handed; it hardcodes no person-specific value.
